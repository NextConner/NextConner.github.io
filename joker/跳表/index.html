¶跳表

¶概念

  1. 前面在学习二分查找的时候，是基于有序数组实现的，因为数组支持随机访问，所有可以通过折半的方式去定位数据；那如果是在链表中想要实现二分查找这样的操作，应该怎么做？需要对链表进行改造，改造之后的链表叫做跳表

  2. 我们知道单链表即使存储数据有序，因为存储方式的问题，查找一个元素的只能顺序遍历，查找的效率并不高，要提高查找效率，就得实现类似二分查找那样可以不断缩小查找范围的存储结构，减少定位元素需要遍历的次数。

  3. 解决方法是对链表建立索引层，索引层的节点来自底层链表中按照指定间隔顺序取出的节点，，索引层的节点保存了指向链表的指针，例如每2个节点取出一个作为索引层的节点；链表 : 1,2,3,4,5,6,7 的索引层就会是: 1,3,5,7，这样查找值为4的节点在索引层中遍历两次定位到该节点在初始链表的3~5之间，然后遍历一次就可以定位到值为4的节点

  4. 图比较直观:

    2. 这里对初始链表建立了一层索引，每两个节点取出一个作为索引节点，假设定位元素5，在索引层遍历两次，查找到值为 5 的节点位于 4~7 之间，转到初始链表中进行查询，再遍历一次就定位到了元素位置，相比之下效率有所提升

  5. 从上图的结构也可以看出，对于一条很长的链表，建立一层索引并不能有效提高查找效率，因此跳表是具备多层索引结构的，最后一层索引的节点数为2，刚好将链表分为前后两部分

  6. 关于索引层的节点数量，假设长度为 n 链表 ，按照两个节点取出一个索引节点的方式，第一层索引 n/2 个节点，第二层有 n/4 个，第三层有 n/8 个节点…最后一层有2个节点，则第K层的索引节点数量表示为 : K = n/(2^k)

¶跳表的查询优化

  1. 前面分析了跳表的实现原理和可能带来的查询性能提升，接下来看下实际跳表查询所需要的时间复杂度
  2. 对于 n 长的链表，每隔 2 个节点取出一个索引节点，建立了 K 层索引 ，每层索引的节点数量为 n/(2^k) ，分解一下查找指定元素值为 J 的过程:
    1. 在第K层索引中比较链表头节点和中央节点确定 J 的位置是在初始链表的 0 ~ mid 之间还是 mid ~ n 之间
    2. 在第 K -1 层索引中确定 J 位于 0~ mid/2 ，mid/2 ~ mid 或 mid ~ (3/4)*mid , (3/4)*mid ~ n 之间的某个区域
    3. 可以看出每一层都是遍历了头尾和中间的节点确定查找的数据区间，也就是每一层都比较了3次，当然了这是基于每两个节点取出一个索引节点的划分结果
    4. 而每层的节点数量为 n/(2^k) ，最后一层的节点数为2 ，即 : n/(2^k) = 2 ， k = log2(n-1) ，加上初始链表，整个跳表高度为 log2n ,也就是需要查找 log2n层， 每一层需要遍历 3 个节点，总的时间复杂度就是 O(m*log2n) = O(logn)，所以跳表查找的时间复杂度和数组二分查找的时间复杂度一样
